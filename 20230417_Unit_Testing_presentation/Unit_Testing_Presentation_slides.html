Unit testing¶

  • Perform smaller test; checking one component at a time
  • Often check only one method or function
  • A kind of white box testing: inspect and verify the inner workings of a software

Integration testing¶

  • Test multiple components at once, e.g. classes, functions and modules
  • Check if the data flows correctly between different components
  • Usually the individual modules are already tested with unit testing methods

def lint_package(package: Path) -> Literal['valid', 'invalid', 'needs review']:
    """Run all linting tests against a package"""
    result = 'valid'

    less_strict_tests = [
        metadata_folder_has_one_or_less_file,
        metadata_file_has_valid_filename,
        package_has_no_hidden_file
    ]

    for test in less_strict_tests:
        if not test(package):
            result = 'needs review'

    strict_tests = [
        package_has_valid_name,
        package_has_valid_subfolder_names,
        objects_folder_has_no_access_folder,
        metadata_folder_is_flat,
        objects_folder_has_file,
        package_has_no_bag,
        package_has_no_zero_bytes_file
    ]

    for test in strict_tests:
        if not test(package):
            result = 'invalid'

    return result

Context of our use¶

  • Creating lint_er.py
    • A linter is a static program that catches errors, bugs and flags potential problems in the source code
    • In our context, lint_er.py is a Python script that confirms each Electronic Record (ER) package conforms to the structure expected by the packaging and ingest processes
    • It lints er packages, hence the script name
    • lint_er.py performs many checks. Therefore, testing the script is ideal

Development process of lint_er.py¶

  1. List all the issues we would like to catch
  2. Determine how serious if the issue exists, namely, throwing a warning or an error (The degree is user-defined)
  3. Start writing functions in lint_er.py
  4. Start writing test_lint_er.py, utilizing the pytest module

You can jump between writing the functions and tests.
Writing tests may change how you write functions.

Issue examples¶

  • Top level folder name has to conform to M\d+(ER|DI|EM)\d+: error
  • Second level folders must have objects and metadata folder: error
  • metadata folder is flat:
    • submissionDocumentation folder exists: error
    • submissionDocumentation folder has file(s) in it: error
  • Hidden system files exist in the package: warning

flatten_er_metadata_folder.py¶

  1. Good practice script, since it is relatively small
  2. Logic of the script:
    1. Go through every package (e.g. M1234_ER_0001)
    2. Get the path for submissionDocumentation
    3. Check if the subissionDocumentation folder has any files
      1. if yes, move it / them up one level
      2. if not, delete the folder
  • Delete function should be reusable, as it can delete the folder after moving file(s)
  • We don't need to check too many things, since they will be checked by the linter, e.g. file naming convention

def get_submissionDocumentation_path(package: Path) -> Path | None:
    """Get submissionDocumentation folder path under metadata"""
    expected_path = package / 'metadata' / 'submissionDocumentation'
    
    if expected_path.is_dir():
        return expected_path
    else:
        return None

# This is the test for move_subdoc_files_to_mdfolder

def test_move_subdoc_files_to_mdfolder(common_package):
    """Test that move_subdoc_files_to_mdfolder moves file(s) from submissionDocumentation
    up to directly under metadata folder: (1) there's file under metadata;
    (2) there's no file in submissionDocumentation"""
    
    for subdoc in common_package.rglob('submissionDocumentation'):
        subdoc_file_ls = [x for x in subdoc.iterdir() if x.is_file()]
        # x is Path object that's under subdoc folder AND it is a file, not directory
        # we know this would be M12345_ER_0001.csv because we set the fixture

    flatten_md.move_subdoc_files_to_mdfolder(subdoc_file_ls)

    for md_path in common_package.rglob('metadata'):
        md_file_ls = [x for x in md_path.iterdir() if x.is_file()]
        for subdoc in md_path.glob('submissionDocumentation'):
            subdoc_ls = [x for x in subdoc.iterdir()]

    if len(md_file_ls) > 0 and len(subdoc_ls) == 0:
        result = True
    else:
        result = False

    assert result == True
